代理模式

2020-12-21
目录
    • 静态代理
    • 动态代理

为某个对象提供一个代理，以控制对这个对象的访问。 代理类和委托类有共同的父类或父接口，这样在任何使用委托类对象的地方都可以用代理对象替代。代理类负责请求的预处理、过滤、将请求分派给委托类处理、以及委托类执行完请求后的后续处理。

一个利用代理模式构建的程序中，包含三种元素：代理接口（Subject）、代理类（ProxySubject）、委托类（RealSubject）。

模拟一个场景： 委托类要处理一项耗时较长的任务，客户类需要打印出执行任务消耗的时间。解决这个问题需要记录任务执行前时间和任务执行后时间，两个时间差就是任务执行消耗的时间。

静态代理

由程序员创建或工具生成代理类的源码，再编译代理类。所谓静态也就是在程序运行前就已经存在代理类的字节码文件，代理类和委托类的关系在运行前就确定了。

直接上代码：

  1. 代理接口
 /**
 * 代理接口，需要代理的任务
 */
public interface Subject {
    void doTask();
}
  1. 委托类，具体处理业务
/**
 * 具体的任务实现， 实现方式1
 */
public class RealSubject1 implements Subject{
    @Override
    public void doTask() {
        System.out.println("DO TASK - RealSubject1 -> Start");
        // 在这里处理了很多的事情 ...
        System.out.println("DO TASK - RealSubject1 -> End");
    }
}
/**
 * 具体的任务实现， 实现方式2
 */
public class RealSubject2 implements Subject{
    @Override
    public void doTask() {
        System.out.println("DO TASK - RealSubject2 -> Start");
        // 在这里处理了很多的事情 ...
        System.out.println("DO TASK - RealSubject2 -> End");
    }
}
  1. 静态代理类
/**
 * 静态代理类， 需要有一个 被代理的 委托类 的引用 private Subject subject 来操作被代理的 委托类
 */
public class SubjectProxy implements Subject{

    private Subject subject;

    public SubjectProxy(Subject subject){
        this.subject = subject;
    }

    @Override
    public void doTask() {
        System.out.println("Subject Proxy -> Start");
        subject.doTask();
        System.out.println("Subject Proxy -> End");
    }
}
  1. 客户类
/**
 * 客户端可以根据自身需要调用代理类， 
 * 通过代理类， 委托类的具体实现对用户来说是屏蔽的
 * 委托类可以有不同的实现方式
 * 用户只需要调用代理即可
 */
public class Client {
    public static void main(String[] args) {
        Subject subject1 = new RealSubject1();
        SubjectProxy subjectProxy = new SubjectProxy(subject1);
        subjectProxy.doTask();

        System.out.println();

        Subject subject2 = new RealSubject2();
        subjectProxy = new SubjectProxy(subject2);
        subjectProxy.doTask();
    }
}

动态代理

动态代理类的源码是在程序运行期间由JVM根据反射等机制动态的生成，所以不存在代理类的字节码文件。代理类和委托类的关系是在程序运行时确定。
